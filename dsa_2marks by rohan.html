UNIT I Data Structures
Q1 Define Data Structure. ▶

A data structure is a way of organizing and storing data in a computer so it can be accessed and modified efficiently. It defines the relationship between data and the operations that can be performed on it.

Example
Array: [10, 20, 30, 40] → Data is stored in a sequence, each element accessed by index. Types: Array, Stack, Queue, Tree, Graph, etc.
Q2 What is a Sparse Matrix? ▶

A sparse matrix is a matrix where most of the elements are zero. Instead of storing all zeros (which wastes memory), only non-zero values are stored along with their row and column positions.

Example
Matrix: Stored as (row, col, value): 0 0 3 0 (0, 2, 3) 0 0 0 0 → (2, 1, 7) 0 7 0 0 → Only 2 values stored instead of 12!
Q3 Define Stack and mention two applications. ▶

A stack is a linear data structure that follows the LIFO (Last In, First Out) principle — the last element inserted is the first one removed. Operations: push (insert) and pop (remove).

Applications
1. Undo feature in text editors (Ctrl+Z) → Last action done is undone first. 2. Expression evaluation → Converting infix to postfix uses a stack.
Q4 What is a Queue? ▶

A queue is a linear data structure that follows the FIFO (First In, First Out) principle — the first element inserted is the first one removed. It has two ends: front (for removal) and rear (for insertion).

Example
Queue: Front → [10, 20, 30] ← Rear → 10 was added first, so it leaves first. Real life: People waiting in a ticket line.
Q5 Define Linked List. ▶

A linked list is a linear data structure where each element (called a node) contains two parts: the data and a pointer to the next node. Nodes are not stored in contiguous memory locations.

Example
[10 | •]→[20 | •]→[30 | NULL] Node1 Node2 Node3 Each node points to the next one in the chain.
Q6 Differentiate between Stack and Queue. ▶
Comparison
STACK QUEUE ───────────────────────────────────── LIFO (Last In First Out) FIFO (First In First Out) One end (top) Two ends (front & rear) push / pop enqueue / dequeue Example: Undo in editor Example: Printer job queue
Q7 What is a Circular Linked List? ▶

A circular linked list is a linked list in which the last node's pointer points back to the first node instead of NULL, forming a circle. There is no beginning or end.

Example
[10]→[20]→[30] ↑_______________↓ The last node (30) points back to 10. Used in: Round-robin CPU scheduling.
Q8 What is a Doubly Linked List? ▶

A doubly linked list is a linked list where each node has two pointers: one pointing to the next node and one pointing to the previous node. You can traverse in both directions.

Example
NULL←[10]⇄[20]⇄[30]→NULL Each node has: prev | data | next Advantage: Can move forward AND backward.
UNIT II Sorting & Searching
Q9 What is Sorting? ▶

Sorting is the process of arranging data in a specific order — either ascending or descending. It makes searching faster and data easier to read.

Example
Unsorted: [40, 10, 30, 20] Sorted (ascending): [10, 20, 30, 40] Algorithms: Bubble Sort, Quick Sort, Merge Sort
Q10 What is Searching? ▶

Searching is the process of finding a specific element (called the key) in a collection of data. It returns the position of the element if found, otherwise indicates it's not present.

Example
Data: [10, 20, 30, 40] Search for 30 → Found at index 2 Types: Linear Search, Binary Search
Q11 Define Linear Search. ▶

Linear search checks each element one by one from the beginning until the target is found or the list ends. It works on both sorted and unsorted data. Time complexity: O(n).

Example
Array: [5, 3, 8, 1, 9] Search: 8 Step 1: 5 == 8? No Step 2: 3 == 8? No Step 3: 8 == 8? Yes! Found at index 2.
Q12 Define Binary Search. ▶

Binary search works on sorted arrays. It repeatedly divides the search range in half — if the middle element is the target, done; else search the left or right half. Time complexity: O(log n).

Example
Array: [1, 3, 5, 7, 9] Search: 7 Mid = 5. 7 > 5 → search right half [7, 9] Mid = 7. Found! Much faster than linear.
Q13 What is Hashing? ▶

Hashing is a technique to convert a key into an index (called a hash code) using a hash function, so data can be stored and retrieved in O(1) constant time. The storage is called a hash table.

Example
Key: "apple" → hash("apple") → index 3 Store "apple" at table[3] Later: find "apple" instantly at table[3].
Q14 What is a Hash Function? ▶

A hash function is a mathematical formula that converts a key into a valid index for the hash table. A good hash function distributes keys evenly and minimizes collisions.

Example
Hash function: h(key) = key % table_size Key = 25, Table size = 10 h(25) = 25 % 10 = 5 → stored at index 5
UNIT III Algorithms
Q15 What is Garbage Collection? ▶

Garbage collection is the automatic process of reclaiming memory that is no longer being used by a program. It frees up memory occupied by objects/nodes that have no reference pointing to them.

Example
Node A was deleted from a linked list. Its memory is still occupied. Garbage collector finds it → frees it → Available for new use. Used in: Java, Python (automatic GC).
Q16 What is Compaction? ▶

Compaction is a memory management technique where all active/used memory blocks are moved together (compacted) to eliminate fragmentation and create one large block of free memory.

Example
Before Compaction: [Used][Free][Used][Free][Used] After Compaction: [Used][Used][Used][Free Free ] → All free memory is now together.
Q17 Define Infix Expression. ▶

An infix expression is the standard way of writing arithmetic expressions where the operator is placed between the operands. Parentheses and precedence rules are used to determine the order of evaluation.

Example
A + B → operator (+) between A and B (A + B) * C → normal math notation we use daily Needs precedence rules to evaluate correctly.
Q18 Define Postfix Expression. ▶

A postfix expression (also called Reverse Polish Notation) places the operator after both operands. No parentheses or precedence rules are needed — computers evaluate it easily using a stack.

Example
Infix: A + B → Postfix: A B + Infix: (A + B)*C → Postfix: A B + C * Evaluation: Read left to right, use a stack.
UNIT IV Trees
Q19 Define Tree. ▶

A tree is a non-linear hierarchical data structure made of nodes connected by edges. It has one root node at the top, and every other node has exactly one parent. No cycles exist.

Example
A ← Root / \ B C ← Children of A / \ D E ← Children of B (Leaf nodes)
Q20 What is a Binary Tree? ▶

A binary tree is a tree in which each node has at most two children: a left child and a right child. It is the most commonly used tree structure.

Example
1 / \ 2 3 / \ 4 5 Each node has maximum 2 children.
Q21 What is a Binary Search Tree (BST)? ▶

A BST is a binary tree with a special rule: for every node, all values in the left subtree are smaller, and all values in the right subtree are greater. This makes searching very efficient.

Example
50 / \ 30 70 / \ / \ 20 40 60 80 Search 60: 60<50? No → go right → 60<70? Yes → go left → Found!
Q22 What is an AVL Tree? ▶

An AVL tree is a self-balancing BST where the difference between heights of left and right subtrees (called the balance factor) is at most 1. If unbalanced, rotations are done to fix it.

Example
Balance Factor = Height(left) - Height(right) Allowed values: -1, 0, +1 If BF = 2 or -2 → tree rotates to rebalance. Ensures O(log n) operations always.
Q23 What is a B+ Tree? ▶

A B+ tree is a balanced multi-level tree used in databases. All data is stored in the leaf nodes, and leaf nodes are linked together. Internal nodes only store keys for routing/searching.

Example
Internal nodes: [10 | 20 | 30] (only keys) Leaf nodes: [5,8]→[10,15]→[20,25]→[30,35] └─── all linked ──────────────┘ Used in: Database indexing (MySQL, etc.)
Q24 Define Leaf Node. ▶

A leaf node is a node in a tree that has no children. It is the bottommost node and represents the end of a branch. Also called an external node or terminal node.

Example
A / \ B C ← C is a leaf (no children) / \ D E ← D and E are leaf nodes
UNIT V Graphs
Q25 What is a Graph? ▶

A graph is a non-linear data structure consisting of vertices (nodes) and edges (connections between nodes). Unlike trees, graphs can have cycles and a node can connect to any other node.

Example
Vertices: A, B, C, D Edges: A-B, A-C, B-D, C-D A — B | | C — D Types: Directed, Undirected, Weighted
Q26 Define BFS (Breadth First Search). ▶

BFS visits all nodes level by level — first the starting node, then its direct neighbors, then their neighbors, and so on. It uses a Queue to keep track of nodes to visit.

Example
Graph: A — B — D | C — E BFS from A: A → B → C → D → E (Level 0: A, Level 1: B,C, Level 2: D,E) Used in: Shortest path, social networks.
Q27 Define DFS (Depth First Search). ▶

DFS visits nodes by going as deep as possible along each branch before backtracking. It uses a Stack (or recursion) to track the path.

Example
Graph: A — B — D | C — E DFS from A: A → B → D → backtrack → C → E Goes deep first, then comes back. Used in: Maze solving, cycle detection.
Q28 What is Topological Sorting? ▶

Topological sorting is a linear ordering of vertices in a directed acyclic graph (DAG) such that for every directed edge from U to V, U comes before V in the ordering. Used for task scheduling.

Example
Tasks: "Get up" → "Brush" → "Eat" → "Go to college" Topological order: Get up, Brush, Eat, College Rule: A task must appear before tasks it leads to. Works only on DAGs (no cycles).
Q29 What is a Path Matrix? ▶

A path matrix (also called a reachability matrix) is a matrix where entry P[i][j] = 1 if there is a path from vertex i to vertex j, and 0 otherwise. It is derived from the adjacency matrix.

Example
Graph: A→B, B→C (A can reach C via B) Path Matrix: A B C A [ 1 1 1 ] ← A reaches A,B,C B [ 0 1 1 ] ← B reaches B,C C [ 0 0 1 ] ← C reaches only itself
